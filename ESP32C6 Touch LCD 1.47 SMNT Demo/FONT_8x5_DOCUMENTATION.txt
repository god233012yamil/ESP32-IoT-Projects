HOW TO USE IN YOUR PROJECT
--------------------------

Step 1: Add font_5x8.h to your project
---------------------------------------

Copy font_5x8.h to your main/ directory:

  esp32c6_wifi_time/
  ├── main/
  │   ├── main.c
  │   ├── font_5x8.h  ← Add this file here
  │   └── CMakeLists.txt
  └── ...

Step 2: Include the header in main.c
-------------------------------------

At the top of main.c, replace the old font array with:

  #include "font_5x8.h"

Step 3: Update char_to_index() function
----------------------------------------

Replace the old char_to_index() function with:

  static int char_to_index(char c) {
      if (c < 32 || c > 126) {
          return 0;  // Return space for invalid characters
      }
      return c - 32;
  }

That's it! Now you can display ANY printable ASCII character.

FONT FORMAT DETAILS
-------------------

Data Structure:
  - Array of 95 characters
  - Each character is 5 bytes
  - Each byte represents one column (5 pixels wide)
  - Each bit in the byte represents one pixel (8 pixels tall)

Pixel Mapping:
  Bit 0 = Top pixel
  Bit 1 = Second pixel
  ...
  Bit 7 = Bottom pixel

Example - Letter 'A' (ASCII 65):
  {0x7E, 0x11, 0x11, 0x11, 0x7E}
  
  Column 0: 0x7E = 01111110
  Column 1: 0x11 = 00010001
  Column 2: 0x11 = 00010001
  Column 3: 0x11 = 00010001
  Column 4: 0x7E = 01111110
  
  Visual representation:
    .####.
    #....#
    #....#
    ######
    #....#
    #....#
    #....#
    ......

TROUBLESHOOTING
---------------

Problem: Character displays as space
Solution: Character may be < 32 or > 126
         Check your string for control characters

Problem: Garbage characters displayed
Solution: Verify you're using the new char_to_index()
         Make sure c - 32 is the correct offset

Problem: Lowercase not working
Solution: Confirm you're using the new font_5x8.h
         Old font didn't have lowercase

Problem: Special symbols not displaying
Solution: Check ASCII value is 32-126
         Some symbols may need escape sequences in C strings
